Nakima
Jun 2014 – Jun 2016

Nakima es una empresa que empezó con temas bursátiles, pero se fue moviendo a consultoría. Mi responsabilidad aquí se basaba en ser arquitecto de software para crear una plantilla que nos garantizara la mayor velocidad en la entrega del producto sin lastimar la calidad del desarrollo, la principal metodología era programación orientada a servicios. Para ello mis responsabilidades eran:

  • Responsable de creación de bundles con Symfony 3 para la optimización de desarrollo proyectos.
  • Responsable de backend de varios proyectos, entre ellos a destacar:
    • Nexiona (Android): Programación de gafas Vucix para retransmisión de video unidireccional, y audio bidireccional.
    • El Canal Audiodescrito: Esta web ya no existe. Esta web tiene como objetivo ser Netflix para la gente ciega, de tal manera podrían reproducir películas audiodescritas, toda la UX estaba dedicada para que la gente ciega pudiera navegar por esta.
    • Joyer: Esta plataforma permite a los médicos ofrecer tablas de ejercicios dinámicas a los pacientes de recuperación en función del feedback que daban los usuarios, por ejemplo, el grado de dolor que había sufrido en hacer el ejercicio, el tiempo dedicado en conseguirlo o el porcentaje del ejercicio que ha logrado realizar entre otros. El gran reto de este proyecto ha sido que los doctores que nos pidieron este proyecto no saben programar, sin embargo, se montaron unos Excel que dado la entrada que daba el usuario, les daba los datos del siguiente ejercicio que el paciente debería realizar, así que programé un compilador que transforma los Excel de los doctores a código PHP, de tal manera que la API permitía la actualización de los Excel y usar la respuesta para devolverla mediante la API. Aunque se puede usar los Excel directamente desde PHP, el compilador redujo los tiempos de respuesta de 10 minutos de media a tan solo algunos milisegundos.
    • Quiroo: Esta web ya no existe: Esta plataforma permitía comunicar a diferentes doctores de manera segura y confidencial los documentos médicos. El servidor solo guarda los ficheros cifrados, pero las claves las guardan los doctores y usuarios, todo el tema de cifrado y descifrado se hace en local con JavaScript.
    • Lasvi: Es una plataforma bursátil donde Carlos Lavignes (cliente) puede visualizar su algoritmo de manera rápida. El reto era conseguir los datos, junto al desarrollador de sistemas creamos un puente que comunicaba PHP con Python con un cliente en Java para obtener los datos bursátiles a tiempo real, Debido a la cantidad de datos que existían Doctrine era un cuello de botella y cree un ORM optimizado para la funcionalidad, haciendo que la carga pasará de 15 minutos a pocos segundos sin aumentar el coste de servidores.
